# PDF 入力値のサニタイズ スペック

## 目的

ユーザー入力値を PDF に出力する前にサニタイズし、Prawn のインラインフォーマッティングタグによる意図しないレイアウト崩れを防止する。

## 背景

- `lib/pdf/notification_pdf/post_pdf.rb` の `create_form` / `create_stamps` で `record.myname` 等のユーザー入力を `text_box` に直接渡している
- Prawn の `text_box` はデフォルトで `inline_format: false` のため、`<b>` 等のタグはそのまま文字列として表示される（深刻な脆弱性ではない）
- ただし、将来的に `inline_format: true` が追加された場合のリスク軽減と、防御的プログラミングの観点からサニタイズを導入する

## 技術スタック・制約

- Rails 7.1.x / Ruby 3.3.6
- Prawn / Prawn-Table
- gem 追加なし

## スコープ

### 範囲内

- `RecordsForm` にサニタイズメソッドを追加し、`<` `>` をエスケープ
- 既存テストが通ること

### 範囲外

- `inline_format: true` の使用有無の変更
- PDF レイアウトの変更

## タスク分解

### フェーズ1: サニタイズ実装

1. `RecordsForm` にコールバックで文字列属性をサニタイズ（`<` → `&lt;`, `>` → `&gt;`）
2. テスト追加
3. 品質ゲート実行
